# Лабораторна робота №2 — Ключові сценарії та зміни даних

## Сценарій 1: Створення профілю користувача

**Вхідні дані:** `name`, (опц.) `weight`, `height`, `goal`  
**Кроки:**

1. Client відправляє `POST /users`
2. API валідуює DTO
3. Users module створює запис User  
   **Результат:** новий `USER` з `id`

**Зміни в даних:** `USER += 1`

---

## Сценарій 2: Оновлення профілю

**Вхідні дані:** `userId`, поля для оновлення  
**Кроки:**

1. Client відправляє `PATCH /users/:id`
2. API валідуює DTO
3. Users module оновлює профіль  
   **Результат:** оновлений `USER`

**Зміни в даних:** оновлення полів `USER`

---

## Сценарій 3: Додавання тренування

**Вхідні дані:** `userId`, `type`, `durationMin`, (опц.) `notes`  
**Кроки:**

1. Client відправляє `POST /workouts`
2. API валідуює DTO (`durationMin > 0`)
3. Workouts module перевіряє існування користувача
4. Створюється `WORKOUT` зі `status = ACTIVE`  
   **Результат:** новий `WORKOUT`

**Зміни в даних:** `WORKOUT += 1`

---

## Сценарій 4: Завершення тренування

**Вхідні дані:** `workoutId`, `status = DONE`, (опц.) `notes`  
**Кроки:**

1. Client відправляє `PATCH /workouts/:id`
2. Workouts module знаходить тренування
3. Оновлює `status`, додає `notes`  
   **Результат:** `WORKOUT.status = DONE`

**Зміни в даних:** оновлення `WORKOUT`

---

## Сценарій 5: Перегляд історії тренувань

**Вхідні дані:** `userId`  
**Кроки:**

1. Client відправляє `GET /workouts?userId=...`
2. Workouts module повертає список тренувань користувача  
   **Результат:** список `WORKOUT[]`

**Дані/агрегації (майбутнє):**

- сума `durationMin` за період
- кількість тренувань за `type`
- кількість завершених/активних тренувань
